# pega-notebooklm-guia
Este projeto foi desenvolvido como parte do desafio da DIO com foco no uso da Inteligência Artificial.  O objetivo deste caderno temático é consolidar os estudos para a certificação Pega System Architect (SA24), utilizando o NotebookLM para organização de conhecimento, geração de resumos, perguntas inteligentes e revisões assistidas por IA.
Objetivos de Estudo

- Revisar conceitos fundamentais da plataforma Pega
- Consolidar conhecimentos sobre Case Management
- Aprender boas práticas de modelagem de dados
- Melhorar a preparação para entrevistas técnicas
- Criar um material reutilizável para futuras revisões

Miniguia de Estudos

 Case Lifecycle

O ciclo de vida de um case representa o fluxo completo de um processo de negócio dentro do Pega.

Principais etapas:
- Create
- Process
- Resolve

Data Pages

Data Pages são estruturas reutilizáveis usadas para carregar dados no Pega.
Tipos:
- Thread
- Requestor
- Node


 Data Transform

Utilizado para:
- copiar valores
- manipular propriedades
- preparar dados
Evita uso excessivo de Activities.

Glossário

| Conceito | Definição |
|---|---|
| Case | Processo de negócio no Pega |
| Data Page | Estrutura de carregamento de dados |
| Flow | Fluxo do processo |
| Assignment | Tarefa atribuída |
| SLA | Controle de prazo |
